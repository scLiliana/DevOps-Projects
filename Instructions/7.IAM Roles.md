## Role para instancias EC2 (Nginx + Tomcat)

**Pre-requisitos:** [[3.Función para guardar el variables en un documento]]
Este rol permite a las instancias EC2 leer/escribir en S3 (para logs y artefactos), y usar SSM y CloudWatch.

> ⚠️ El S3 bucket se crea en el **Paso [[8.S3 Bucket (para logs de Tomcat)]]**. Si aún no lo has creado, define el nombre ahora y úsalo de forma consistente en ambos pasos.

### Crear el nombre del Bucket

```bash
# Definir el nombre del bucket (debe ser único globalmente en AWS)

S3_BUCKET_NAME="devops-project-01-logs-$(aws sts get-caller-identity \
    --query Account \
    --output text)"
    
save_id "S3_BUCKET_NAME" "$S3_BUCKET_NAME"
```

### Recuperar el Account ID de aws

```bash
# Obtener el Account ID

ACCOUNT_ID=$(aws sts get-caller-identity \
    --query Account \
    --output text)
    
save_id "ACCOUNT_ID" "$ACCOUNT_ID"
```

### Crear el Rol y trust policy para EC2

```bash
# Crear el rol con trust policy para EC2

aws iam create-role \
    --role-name EC2AppRole \
    --assume-role-policy-document '{
        "Version": "2012-10-17",
        "Statement": [{
            "Effect": "Allow",
            "Principal": { "Service": "ec2.amazonaws.com" },
            "Action": "sts:AssumeRole"
        }]
    }'
echo "IAM Role EC2AppRole creado"
```

### Crear el Rol y trust policy para S3

**Pre-requisitos:**```$S3_BUCKET_NAME```
```bash
# Crear la política personalizada para S3

POLICY_ARN=$(aws iam create-policy \
    --policy-name AppS3Policy \
    --policy-document "{
        \"Version\": \"2012-10-17\",
        \"Statement\": [{
            \"Effect\": \"Allow\",
            \"Action\": [
                \"s3:GetObject\",
                \"s3:PutObject\",
                \"s3:ListBucket\"
            ],
            \"Resource\": [
                \"arn:aws:s3:::${S3_BUCKET_NAME}\",
                \"arn:aws:s3:::${S3_BUCKET_NAME}/*\"
            ]
        }]
    }" \
    --query "Policy.Arn" \
    --output text)
    
save_id "POLICY_ARN" "$POLICY_ARN"
```
**Pre-requisitos:**```$POLICY_ARN```
```bash
# Adjuntar la política S3 al rol

aws iam attach-role-policy \
    --role-name EC2AppRole \
    --policy-arn $POLICY_ARN
```

### Adjuntar políticas gestionadas por AWS

```bash
# Adjuntar políticas gestionadas por AWS para SSM y CloudWatch

aws iam attach-role-policy \
    --role-name EC2AppRole \
    --policy-arn arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore

aws iam attach-role-policy \
    --role-name EC2AppRole \
    --policy-arn arn:aws:iam::aws:policy/CloudWatchAgentServerPolicy

echo "Políticas adjuntadas al rol"
```

### Instance Profile y asociar el rol

```bash
# Crear Instance Profile y asociar el rol

aws iam create-instance-profile \
    --instance-profile-name EC2AppProfile

aws iam add-role-to-instance-profile \
    --instance-profile-name EC2AppProfile \
    --role-name EC2AppRole

echo "Instance Profile EC2AppProfile creado y asociado"
```

### Verificación

```bash
# Verificar que todo quedó bien

aws iam get-instance-profile \
    --instance-profile-name EC2AppProfile \
    --query "InstanceProfile.{Profile:InstanceProfileName, Rol:Roles[0].RoleName}" \
    --output table
```


## 6.2 Relación de recursos

```
EC2 (Nginx / Tomcat)
    └── Instance Profile: EC2AppProfile
            └── IAM Role: EC2AppRole
                ├── AppS3Policy                  → s3:GetObject, PutObject en tu bucket
                ├── AmazonSSMManagedInstanceCore → acceso via Systems Manager
                └── CloudWatchAgentServerPolicy  → enviar métricas a CloudWatch
```
