## CronJob para enviar logs de Tomcat a S3

Conéctate a cada instancia Tomcat via Bastion y ejecuta:

```bash
# Desde Bastion → instancia Tomcat
IP_TOMCAT=$(aws ec2 describe-instances \
    --filters "Name=tag:Name,Values=tomcat-asg" \
              "Name=instance-state-name,Values=running" \
    --query "Reservations[0].Instances[0].PrivateIpAddress" \
    --output text --region us-east-1)

ssh -i ~/.ssh/DevOpsKeyPair.pem ec2-user@$IP_TOMCAT
```

Dentro de la instancia Tomcat:

```bash
# Obtener el nombre del bucket automáticamente
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
S3_BUCKET_NAME="devops-project-01-logs-${ACCOUNT_ID}"

# Verificar acceso al bucket (el IAM Role debe permitirlo)
aws s3 ls s3://$S3_BUCKET_NAME

# Crear el script de envío de logs
sudo tee /usr/local/bin/push-logs-s3.sh << 'EOF'
#!/bin/bash
BUCKET="devops-project-01-logs-$(aws sts get-caller-identity --query Account --output text)"
DATE=$(date +%Y-%m-%d)
LOG_DIR="/opt/tomcat/logs"

aws s3 cp $LOG_DIR/ s3://$BUCKET/tomcat-logs/$DATE/ \
    --recursive --region us-east-1

# Eliminar logs locales viejos (mayores a 7 días)
find $LOG_DIR -name "*.log" -mtime +7 -delete
echo "Logs enviados a s3://$BUCKET/tomcat-logs/$DATE/"
EOF

sudo chmod +x /usr/local/bin/push-logs-s3.sh

# Probar el script manualmente antes de configurar el cron
sudo /usr/local/bin/push-logs-s3.sh

# Configurar el CronJob (se ejecuta a medianoche todos los días)
# Agrega esta línea en el editor que se abre:
# 0 0 * * * /usr/local/bin/push-logs-s3.sh >> /var/log/push-logs-s3.log 2>&1
crontab -e

# Verificar que quedó registrado
crontab -l
```

> ⚠️ Repite este proceso en **cada instancia Tomcat** que esté corriendo. Para automatizarlo en instancias futuras, agrega el script al `tomcat-userdata.sh`.

## Alarma CloudWatch para conexiones de BD

```bash
# Desde tu máquina local

# Crear topic SNS
SNS_TOPIC_ARN=$(aws sns create-topic \
    --name DBAlarmTopic \
    --query "TopicArn" \
    --output text --region us-east-1)
echo "SNS Topic ARN: $SNS_TOPIC_ARN"

# Suscribir tu email (debes confirmar el correo que llega)
aws sns subscribe \
    --topic-arn $SNS_TOPIC_ARN \
    --protocol email \
    --notification-endpoint tu@email.com \
    --region us-east-1

# Crear alarma: dispara cuando conexiones > 100
aws cloudwatch put-metric-alarm \
    --alarm-name "DBConnectionsHigh" \
    --metric-name DatabaseConnections \
    --namespace AWS/RDS \
    --statistic Average \
    --period 300 \
    --threshold 100 \
    --comparison-operator GreaterThanOrEqualToThreshold \
    --evaluation-periods 1 \
    --alarm-actions $SNS_TOPIC_ARN \
    --dimensions Name=DBInstanceIdentifier,Value=prod-mysql \
    --region us-east-1
echo "Alarma CloudWatch creada"
```

## Validación

### Verificar acceso a EC2 via SSM

```bash
# Listar instancias registradas en SSM
aws ssm describe-instance-information \
    --query "InstanceInformationList[*].{ID:InstanceId,Estado:PingStatus,Nombre:ComputerName}" \
    --output table --region us-east-1

# Abrir sesión sin SSH
aws ssm start-session --target i-INSTANCE_ID --region us-east-1
```

### Verificar estado del Auto Scaling

```bash
aws autoscaling describe-auto-scaling-groups \
    --auto-scaling-group-names tomcat-asg nginx-asg \
    --query "AutoScalingGroups[*].{Nombre:AutoScalingGroupName,Min:MinSize,Max:MaxSize,Deseado:DesiredCapacity,Instancias:Instances[*].InstanceId}" \
    --output table --region us-east-1
```

## Verificar estado del RDS

```bash
aws rds describe-db-instances \
    --db-instance-identifier prod-mysql \
    --query "DBInstances[0].{Estado:DBInstanceStatus,Endpoint:Endpoint.Address,MultiAZ:MultiAZ}" \
    --output table --region us-east-1
```

### Verificar los NLBs


```bash
aws elbv2 describe-load-balancers \
    --names public-nlb private-nlb \
    --query "LoadBalancers[*].{Nombre:LoadBalancerName,DNS:DNSName,Estado:State.Code}" \
    --output table --region us-east-1
```

### Probar la aplicación

```bash
# Obtener el DNS del NLB público dinámicamente
NLB_FRONTEND_DNS=$(aws elbv2 describe-load-balancers \
    --names public-nlb \
    --query "LoadBalancers[0].DNSName" \
    --output text --region us-east-1)
echo "URL de la aplicación: http://$NLB_FRONTEND_DNS"

# Verificar que responde
curl -I http://$NLB_FRONTEND_DNS
# HTTP/1.1 200 OK → aplicación funcionando
```
