## Configurar Nginx como reverse proxy

El archivo `/etc/nginx/conf.d/app.conf` **no se crea manualmente** — lo genera automáticamente el `nginx-userdata.sh` cuando el ASG lanza cada instancia. El contenido final que queda en cada instancia es:

**Pre-requisitos:**`$NLB_BACKEND_DNS`
```nginx
# Sustituir la variable NLB_BACKEND_DNS por el valor real

upstream backend {
    server NLB_BACKEND_DNS:8080;
}
server {
    listen 80;
    server_name _;
    location / {
        proxy_pass http://backend;
        proxy_set_header Host "backend";
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

> ⚠️ **`proxy_set_header Host "backend"`** — usar el nombre del upstream en lugar de `$host` es importante cuando el NLB backend tiene guiones en su DNS (ej. `internal-xxx.elb.amazonaws.com`). Tomcat puede rechazar la petición con HTTP 400 si recibe el DNS completo del NLB como Host header.

> ⚠️ **Error `duplicate upstream "backend"`** — si Nginx arranca con este error, hay dos bloques `upstream backend` definidos: uno en `/etc/nginx/nginx.conf` y otro en `/etc/nginx/conf.d/app.conf`. Solución: eliminar el bloque `upstream` del `nginx.conf` y dejarlo solo en `app.conf`. Luego ejecutar `sudo nginx -t && sudo systemctl reload nginx`.

> ℹ️ **Warning `conflicting server name "_"`** — aparece cuando hay dos bloques `server` con `server_name _;` (uno en `nginx.conf` y otro en `app.conf`). No impide el funcionamiento pero se puede corregir comentando el bloque `server` del `nginx.conf` y dejando solo el de `app.conf`.

## User Data script para Nginx

El `nginx-userdata.sh` se crea en tu **máquina local** con el DNS del NLB interno y se referencia en el Launch Template. Los archivos de userdata van en `~/devops-scripts/`.

```bash
# Paso 1 — Obtener el DNS del NLB interno (si no lo tienes en la variable)

NLB_BACKEND_DNS=$(aws elbv2 describe-load-balancers \
    --names private-nlb \
    --query "LoadBalancers[0].DNSName" \
    --output text --region us-east-1)
echo "Backend NLB DNS: $NLB_BACKEND_DNS"
```

```bash
# Paso 2 — Crear el script con ese DNS
cat > ~/devops-scripts/nginx-userdata.sh << USERDATA
#!/bin/bash
cat > /etc/nginx/conf.d/app.conf << 'EOF'
upstream backend {
    server ${NLB_BACKEND_DNS}:8080;
}
server {
    listen 80;
    server_name _;
    location / {
        proxy_pass http://backend;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
    }
}
EOF
sudo systemctl reload nginx
USERDATA

```

```bash
# Paso 3 — Verificar que el DNS quedó escrito correctamente

grep "server " ~/devops-scripts/nginx-userdata.sh
# Debe mostrar: server XXXX.elb.us-east-1.amazonaws.com:8080;
```

> ⚠️ Ejecuta los pasos 1 y 2 en la **misma sesión de terminal** para que `$NLB_BACKEND_DNS` esté disponible al crear el archivo.

##  Crear Target Group para Nginx

```bash
TG_NGINX_ARN=$(aws elbv2 create-target-group \
    --name nginx-tg \
    --protocol TCP \
    --port 80 \
    --vpc-id $VPC_PRIMARY_ID \
    --health-check-protocol TCP \
    --health-check-port 80 \
    --query "TargetGroups[0].TargetGroupArn" \
    --output text --region us-east-1)
    
save_id "TG_NGINX_ARN" "$TG_NGINX_ARN"
```

## Crear NLB público (frontend)

### Recuperar ID de subnets públicas si no estan guardados
```bash
SUBNET_PUBLIC_1A_ID=$(aws ec2 describe-subnets \
    --filters "Name=vpc-id,Values=$VPC_PRIMARY_ID" \
              "Name=tag:Name,Values=PublicSubnet-1a" \
    --query "Subnets[0].SubnetId" --output text --region us-east-1)

SUBNET_PUBLIC_1B_ID=$(aws ec2 describe-subnets \
    --filters "Name=vpc-id,Values=$VPC_PRIMARY_ID" \
              "Name=tag:Name,Values=PublicSubnet-1b" \
    --query "Subnets[0].SubnetId" --output text --region us-east-1)
```

### Crear Network Load Balancer

**Pre-requisitos:** `$SUBNET_PUBLIC_1A_ID, $SUBNET_PUBLIC_1B_ID`
```bash
# Crea el NLB y guarda el ARN

NLB_FRONTEND_ARN=$(aws elbv2 create-load-balancer \
    --name public-nlb \
    --subnets $SUBNET_PUBLIC_1A_ID $SUBNET_PUBLIC_1B_ID \
    --type network \
    --scheme internet-facing \
    --tags Key=Name,Value=FrontendNLB \
    --query "LoadBalancers[0].LoadBalancerArn" \
    --output text --region us-east-1)

save_id "NLB_FRONTEND_ARN" "$NLB_FRONTEND_ARN"
```

```bash
# Recuperar el DNS

NLB_FRONTEND_DNS=$(aws elbv2 describe-load-balancers \
    --load-balancer-arns $NLB_FRONTEND_ARN \
    --query "LoadBalancers[0].DNSName" \
    --output text --region us-east-1)

save_id "NLB_FRONTEND_DNS" "$NLB_FRONTEND_DNS"
```

```bash
# Crear listener

aws elbv2 create-listener \
    --load-balancer-arn $NLB_FRONTEND_ARN \
    --protocol TCP \
    --port 80 \
    --default-actions Type=forward,TargetGroupArn=$TG_NGINX_ARN \
    --region us-east-1
    
echo "Listener Nginx creado"
```

## Launch Template para Nginx

```bash
# Recuperar ID de la Ngnix AMI

NGINX_AMI=$(aws ec2 describe-images --owners self \
    --filters "Name=name,Values=NginxGoldenAMI" \
    --query "Images[0].ImageId" --output text --region us-east-1)
```

```bash
# Codificar UserData en Base64 (macOS)

NGINX_USERDATA=$(base64 -i ~/devops-scripts/nginx-userdata.sh)

# En Linux usar:
# NGINX_USERDATA=$(base64 -w 0 ~/devops-scripts/nginx-userdata.sh)
```

**Pre-requisitos:**`$NGINX_AMI, $SG_FRONTEND_ID, $NGINX_USERDATA`
```bash
# Crear la Launch Template

aws ec2 create-launch-template \
    --launch-template-name NginxLT \
    --version-description "Nginx Launch Template v1" \
    --launch-template-data "{
        \"ImageId\": \"$NGINX_AMI\",
        \"InstanceType\": \"t3.micro\",
        \"KeyName\": \"DevOpsKeyPair\",
        \"SecurityGroupIds\": [\"$SG_FRONTEND_ID\"],
        \"IamInstanceProfile\": {\"Name\": \"EC2AppProfile\"},
        \"UserData\": \"$NGINX_USERDATA\",
        \"TagSpecifications\": [{
            \"ResourceType\": \"instance\",
            \"Tags\": [{\"Key\": \"Name\", \"Value\": \"nginx-asg\"}]
        }]
    }" \
    --region us-east-1
echo "Launch Template NginxLT creado"
```

## Auto Scaling Group para Nginx

**Pre-requisitos:**`$TG_NGINX_ARN, $SUBNET_PRIVATE_1A_ID,$SUBNET_PRIVATE_1B_ID`
```bash
# Crear el Auto Scaling Group

aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name nginx-asg \
    --launch-template "LaunchTemplateName=NginxLT,Version=\$Latest" \
    --min-size 1 \
    --max-size 3 \
    --desired-capacity 2 \
    --target-group-arns $TG_NGINX_ARN \
    --vpc-zone-identifier "$SUBNET_PRIVATE_1A_ID,$SUBNET_PRIVATE_1B_ID" \
    --health-check-type ELB \
    --health-check-grace-period 300 \
    --tags Key=Name,Value=nginx-asg,PropagateAtLaunch=true \
    --region us-east-1
echo "ASG Nginx creado"
```
