# Security Groups

**Pre-requisitos:** [[4.VPC y Networking]], [[3.Función para guardar el variables en un documento]]

### Diagrama de flujo de los Security Groups

>Internet → FrontendSG (80/443) → BackendSG (8080) → DatabaseSG (3306)→ BastionSG (22) → todos los SGs via SSH

## Security Group Frontend

**Pre-requisitos:** ```$VPC_PRIMARY_ID```
```bash
# Crear el Security Group

SG_FRONTEND_ID=$(aws ec2 create-security-group \
    --group-name FrontendSG \
    --description "Security group for frontend Nginx servers" \
    --vpc-id $VPC_PRIMARY_ID \
    --query "GroupId" \
    --output text \
    --region us-east-1)
    
save_id "SG_FRONTEND_ID" "$SG_FRONTEND_ID"
```

**Pre-requisitos:** ```$SG_FRONTEND_ID```
```bash
# Permitir HTTP desde internet

aws ec2 authorize-security-group-ingress \
    --group-id $SG_FRONTEND_ID \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0 \
    --region us-east-1
```
```bash
# Permitir HTTPS desde internet

aws ec2 authorize-security-group-ingress \
    --group-id $SG_FRONTEND_ID \
    --protocol tcp \
    --port 443 \
    --cidr 0.0.0.0/0 \
    --region us-east-1
```
```bash
# Permitir SSH desde Bastion

aws ec2 authorize-security-group-ingress \
    --group-id $SG_FRONTEND_ID \
    --protocol tcp \
    --port 22 \
    --source-group SG_BASTION_ID \
    --region us-east-1
```

```bash
# Verificar las reglas creadas

aws ec2 describe-security-groups \
    --group-ids $SG_FRONTEND_ID \
    --query "SecurityGroups[0].IpPermissions" \
    --output table --region us-east-1
```

    --group-name BackendSG \
