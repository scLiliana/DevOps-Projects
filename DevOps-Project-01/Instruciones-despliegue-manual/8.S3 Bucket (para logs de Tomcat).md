# S3 Bucket (para logs de Tomcat)

[[1.DevOps-Proyect-01]][[Infra DevOps]][[AWS]]


ℹ️ Si ya definiste `S3_BUCKET_NAME` en el paso [[7.IAM Roles]], reutiliza esa variable. Si empezaste una sesión nueva, recupérala así:

```bash
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)

S3_BUCKET_NAME="devops-project-01-logs-${ACCOUNT_ID}"

echo "Bucket: $S3_BUCKET_NAME"
```

O del documento ```"source ~/aws_ids.env"``` si se esta haciendo uso de [[3.Función para guardar el variables en un documento]]

## Crear Bucket

**Pre-requisitos:** `S3_BUCKET_NAME`
```bash 
# Crear el bucket
aws s3api create-bucket \
    --bucket $S3_BUCKET_NAME \
    --region us-east-1
    
echo "Bucket $S3_BUCKET_NAME creado"
```

## Configurar Bucket

```bash
# Habilitar versionado
aws s3api put-bucket-versioning \
    --bucket $S3_BUCKET_NAME \
    --versioning-configuration Status=Enabled
```

```bash
# Bloquear todo acceso público
aws s3api put-public-access-block \
    --bucket $S3_BUCKET_NAME \
    --public-access-block-configuration \
	BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

```bash
# Crear carpetas base dentro del bucket
aws s3api put-object --bucket $S3_BUCKET_NAME --key tomcat-logs/
aws s3api put-object --bucket $S3_BUCKET_NAME --key artifacts/
```

```bash
# Verificar que el bucket quedó bien configurado
aws s3api get-bucket-versioning --bucket $S3_BUCKET_NAME
aws s3api get-public-access-block --bucket $S3_BUCKET_NAME

echo "Bucket configurado correctamente"
echo "URL: s3://$S3_BUCKET_NAME"
```
