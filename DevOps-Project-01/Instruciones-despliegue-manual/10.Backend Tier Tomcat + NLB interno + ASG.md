## Crear Target Group para Tomcat

Un Target Group (grupo de destino) en la infraestructura de AWS sirve para agrupar recursos backend (instancias EC2, IPs, funciones Lambda o contenedores) y dirigir el tráfico de un Load Balancer (equilibrador de carga) hacia ellos. Permite equilibrar cargas, gestionar la salud de los destinos y escalar automáticamente la infraestructura. 


**Pre-requisitos:** `$VPC_PRIMARY_ID`
```bash
source $IDS_FILE #Para recuperar variables 

TG_TOMCAT_ARN=$(aws elbv2 create-target-group \
    --name tomcat-tg \
    --protocol TCP \
    --port 8080 \
    --vpc-id $VPC_PRIMARY_ID \
    --health-check-protocol TCP \
    --health-check-port 8080 \
    --query "TargetGroups[0].TargetGroupArn" \
    --output text --region us-east-1)
    
save_id "TG_TOMCAT_ARN" "$TG_TOMCAT_ARN"
```

## Crear NLB interno (backend)

### Recuperar ID de subnets  privadas si no estan guardados

**Pre-requisitos:** `$VPC_PRIMARY_ID`
```bash
SUBNET_PRIVATE_1A_ID=$(aws ec2 describe-subnets \
    --filters "Name=vpc-id,Values=$VPC_PRIMARY_ID" \
              "Name=tag:Name,Values=PrivateSubnet-1a" \
    --query "Subnets[0].SubnetId" --output text --region us-east-1)
    
save_id "SUBNET_PRIVATE_1A_ID" "$SUBNET_PRIVATE_1A_ID"
```

```bash

    --query "Subnets[0].SubnetId" --output text --region us-east-1)
    
save_id "SUBNET_PRIVATE_1B_ID" "$SUBNET_PRIVATE_1B_ID"
```

### Crear Network Load Balancer

**Pre-requisitos:** `$SUBNET_PRIVATE_1A_ID, $SUBNET_PRIVATE_1B_ID`
```bash
# Crea el NLB y guarda el ARN

NLB_BACKEND_ARN=$(aws elbv2 create-load-balancer \
    --name private-nlb \
    --subnets $SUBNET_PRIVATE_1A_ID $SUBNET_PRIVATE_1B_ID \
    --type network \
    --scheme internal \
    --tags Key=Name,Value=BackendNLB \
    --query "LoadBalancers[0].LoadBalancerArn" \
    --output text --region us-east-1)
    
save_id "NLB_BACKEND_ARN" "$NLB_BACKEND_ARN"
```

**Pre-requisitos:** `$NLB_BACKEND_ARN`
```bash
# Recuperar el DNS

NLB_BACKEND_DNS=$(aws elbv2 describe-load-balancers \
    --load-balancer-arns $NLB_BACKEND_ARN \
    --query "LoadBalancers[0].DNSName" \
    --output text --region us-east-1)
    
save_id "NLB_BACKEND_DNS" "$NLB_BACKEND_DNS"
```

Un listener de un Network Load Balancer (NLB) en AWS sirve para verificar, gestionar y dirigir el tráfico de red entrante (capa 4, TCP/UDP) hacia los grupos de destino (Target Groups) configurados. Actúa como el punto de entrada que escucha en un puerto específico y protocolo, aplicando reglas para enrutar las conexiones.

**Pre-requisitos:** `$NLB_BACKEND_ARN`
```bash
# Crear listener del NLB backend

aws elbv2 create-listener \
    --load-balancer-arn $NLB_BACKEND_ARN \
    --protocol TCP \
    --port 8080 \
    --default-actions Type=forward,TargetGroupArn=$TG_TOMCAT_ARN \
    --region us-east-1
    
echo "Listener Tomcat creado"
```

## User Data script para Tomcat

```bash
mkdir -p ~/devops-scripts
cat > ~/devops-scripts/tomcat-userdata.sh << 'USERDATA'
#!/bin/bash
# ⚠️ IMPORTANTE: Reemplaza TU_JFROG_EMAIL y TU_JFROG_ACCESS_TOKEN con tus credenciales reales
# JFrog requiere autenticación para descargar artefactos — sin credenciales wget descargará
# una página HTML de login (~12KB) en lugar del WAR real (~37MB)

JFROG_URL="https://devopsproyect01.jfrog.io/artifactory/libs-release-local/com/devopsrealtime/dptweb/1.0/dptweb-1.0.war"
JFROG_USER="TU_JFROG_EMAIL"
JFROG_TOKEN="TU_JFROG_ACCESS_TOKEN"
WAR_DEST="/opt/tomcat/webapps/ROOT.war"

# Eliminar WAR anterior si existe
sudo rm -f $WAR_DEST
sudo rm -rf /opt/tomcat/webapps/ROOT

# Descargar WAR desde JFrog con autenticación
sudo wget --user=$JFROG_USER --password=$JFROG_TOKEN \
    -O $WAR_DEST $JFROG_URL

# Verificar que el WAR es válido (no una página HTML)
WAR_SIZE=$(stat -c%s $WAR_DEST 2>/dev/null || echo 0)
if [ "$WAR_SIZE" -lt 1000000 ]; then
    echo "ERROR: WAR demasiado pequeño ($WAR_SIZE bytes) — probablemente falló la autenticación con JFrog"
    exit 1
fi

echo "WAR descargado correctamente: $(ls -lh $WAR_DEST)"
sudo systemctl restart tomcat
USERDATA

echo "tomcat-userdata.sh creado"
cat ~/devops-scripts/tomcat-userdata.sh
```

`USERDATA` extendido

```bash
mkdir -p ~/devops-scripts
cat > ~/devops-scripts/tomcat-userdata.sh << 'USERDATA'
#!/bin/bash
set -e

# ⚠️ IMPORTANTE: Reemplaza las variables con tus credenciales y endpoints reales
JFROG_USER="TU_JFROG_EMAIL"
JFROG_TOKEN="TU_JFROG_ACCESS_TOKEN"
JFROG_URL="https://devopsproyect01.jfrog.io/artifactory/libs-release-local/com/devopsrealtime/dptweb/1.0/dptweb-1.0.war"
WAR_DEST="/opt/tomcat/webapps/ROOT.war"
RDS_ENDPOINT="TU_RDS_ENDPOINT"
DB_PASSWORD="TU_DB_PASSWORD"

# Limpiar deploy anterior
sudo systemctl stop tomcat || true
sudo rm -f $WAR_DEST
sudo rm -rf /opt/tomcat/webapps/ROOT

# Descargar WAR desde JFrog con autenticación
sudo wget --user=$JFROG_USER --password=$JFROG_TOKEN \
    -O $WAR_DEST $JFROG_URL

# Verificar que el WAR es válido (no una página HTML de error)
WAR_SIZE=$(stat -c%s $WAR_DEST 2>/dev/null || echo 0)
if [ "$WAR_SIZE" -lt 1000000 ]; then
    echo "ERROR: WAR demasiado pequeño ($WAR_SIZE bytes) — fallo de autenticación con JFrog"
    exit 1
fi

echo "WAR descargado correctamente: $(ls -lh $WAR_DEST)"

# Iniciar Tomcat para que extraiga el WAR
sudo systemctl start tomcat
sleep 15

# Corregir application.properties con el endpoint real de RDS
APP_PROPS="/opt/tomcat/webapps/ROOT/WEB-INF/classes/application.properties"
if [ -f "$APP_PROPS" ]; then
    sudo tee $APP_PROPS << EOF
spring.datasource.url = jdbc:mysql://${RDS_ENDPOINT}:3306/UserDB
spring.datasource.username = admin
spring.datasource.password = ${DB_PASSWORD}
spring.datasource.driver-class-name = com.mysql.cj.jdbc.Driver
spring.jpa.hibernate.ddl-auto = update
EOF
    echo "application.properties actualizado con endpoint RDS"
fi

# Instalar cliente MySQL/MariaDB
sudo dnf install -y mariadb105 2>/dev/null || sudo yum install -y mysql

# Crear base de datos solo si no existe
mysql -h ${RDS_ENDPOINT} -u admin -p"${DB_PASSWORD}" --connect-timeout=10 \
    -e "CREATE DATABASE IF NOT EXISTS UserDB;"

# Reiniciar Tomcat con la configuración actualizada
sudo systemctl restart tomcat
echo "Deploy completado exitosamente"
USERDATA

echo "tomcat-userdata.sh creado"
cat ~/devops-scripts/tomcat-userdata.sh
```

## Launch Template para Tomcat

> ℹ️ Launch Templates reemplazaron a Launch Configurations. El UserData debe ir codificado en Base64. 
> - **Linux:** `base64 -w 0 script.sh`
> - **macOS:** `base64 -i script.sh`

```bash
# Recuperar ID de la Tomcat AMI

TOMCAT_AMI=$(aws ec2 describe-images --owners self \
    --filters "Name=name,Values=TomcatGoldenAMI" \
    --query "Images[0].ImageId" --output text --region us-east-1)
```

**Pre-requisitos:** [[10.Backend Tier Tomcat + NLB interno + ASG#User Data script para Tomcat]]
```bash
# Codificar UserData en Base64 (macOS)

TOMCAT_USERDATA=$(base64 -i ~/devops-scripts/tomcat-userdata.sh)

# En Linux usar:
# TOMCAT_USERDATA=$(base64 -w 0 ~/devops-scripts/tomcat-userdata.sh)
```

**Pre-requisitos:**`$TOMCAT_AMI, $SG_BACKEND_ID, $TOMCAT_USERDATA`
```bash
# Crear la Launch Template

aws ec2 create-launch-template \
    --launch-template-name TomcatLT \
    --version-description "Tomcat Launch Template v1" \
    --launch-template-data "{
        \"ImageId\": \"$TOMCAT_AMI\",
        \"InstanceType\": \"t3.micro\",
        \"KeyName\": \"DevOpsKeyPair\",
        \"SecurityGroupIds\": [\"$SG_BACKEND_ID\"],
        \"IamInstanceProfile\": {\"Name\": \"EC2AppProfile\"},
        \"UserData\": \"$TOMCAT_USERDATA\",
        \"TagSpecifications\": [{
            \"ResourceType\": \"instance\",
            \"Tags\": [{\"Key\": \"Name\", \"Value\": \"tomcat-asg\"}]
        }]
    }" \
    --region us-east-1
echo "Launch Template TomcatLT creado"
```

## Auto Scaling Group para Tomcat

**Pre-requisitos:**`$TG_TOMCAT_ARN, $SUBNET_PRIVATE_1A_ID,$SUBNET_PRIVATE_1B_ID`
```bash
# Crear el Auto Scaling Group

aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name tomcat-asg \
    --launch-template "LaunchTemplateName=TomcatLT,Version=\$Latest" \
    --min-size 1 \
    --max-size 3 \
    --desired-capacity 2 \
    --target-group-arns $TG_TOMCAT_ARN \
    --vpc-zone-identifier "$SUBNET_PRIVATE_1A_ID,$SUBNET_PRIVATE_1B_ID" \
    --health-check-type ELB \
    --health-check-grace-period 300 \
    --tags Key=Name,Value=tomcat-asg,PropagateAtLaunch=true \
    --region us-east-1
echo "ASG Tomcat creado"
```


## Crear la base de datos 

Iniciar una instancia Tomcat o usar una que ya se encuentre activa

Remplazar 
+ `prod-mysql.bw3efoo94zed.us-east-1.rds.amazonaws.com`
+ `contrasenaSegura`

```bash
# Crear la base de datos

COMMAND_ID=$(aws ssm send-command \
    --instance-ids $INSTANCIA_TOMCAT \
    --document-name "AWS-RunShellScript" \
    --parameters '{"commands":[
        "sudo dnf install -y mariadb105 2>/dev/null || sudo yum install -y mysql",
        "mysql -h prod-mysql.bw3efoo94zed.us-east-1.rds.amazonaws.com -u admin -pcontrasenaSegura -e \"CREATE DATABASE IF NOT EXISTS UserDB; SHOW DATABASES;\"",
        "sudo systemctl restart tomcat",
        "sleep 25",
        "sudo tail -15 /opt/tomcat/logs/catalina.out"
    ]}' \
    --query "Command.CommandId" \
    --output text --region us-east-1)

echo $COMMAND_ID
sleep 50

aws ssm get-command-invocation \
    --command-id "$COMMAND_ID" \
    --instance-id $INSTANCIA_TOMCAT \
    --query "StandardOutputContent" \
    --output text --region us-east-1
```
